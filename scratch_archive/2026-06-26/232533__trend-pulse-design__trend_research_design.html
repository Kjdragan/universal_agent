Recrafting analyze-google-trends → a prompt-driven trend-pulse

Borrowing the proven last30days pattern · keyless-first · entertaining packaging · 2026-06-26

The skill today does one thing: pull the live Google Trends country board (~10 breakout searches) and write a chat briefing. The recraft keeps that as one signal and wraps it in a broader, prompt-driven trend-research engine — fan out across keyless sources around your topic, score & cluster deterministically, then package the result in a useful/entertaining brief whose shape changes with what you asked for.

1 · What last30days proves (the pattern we steal)

It researches "what people actually say about any topic in the last 30 days" across Reddit, X, YouTube, TikTok, HN, Polymarket, GitHub, web. The reusable spine, all of it zero paid keys for the parts that matter:

  • Intent → template. It classifies the prompt into a QUERY_TYPE (NEWS / RECOMMENDATIONS / COMPARISON / GENERAL…) and emits a different output shape per type. Your "topic + intent" maps straight onto this.
  • Keyless multi-source fan-out. Reddit (RSS, no key), Hacker News (Algolia API), YouTube (yt-dlp), web (WebSearch), Polymarket (free API) all work with no paid keys. X / TikTok / IG are optional paid add-ons behind graceful "unlock for more coverage" nudges.
  • Deterministic score → LLM synthesize. Code fetches, scores by recency + per-source engagement, and clusters items into cross-platform stories with uncertainty tags; the LLM only writes the narrative. A "trend" is corroborated, not a single board's guess.
  • Packaging devices that make it fun: a status badge, bold-headline paragraphs each anchored to an inline citation, a numbered KEY PATTERNS list, verbatim high-upvote "best takes" quotes, head-to-head tables for comparisons, and an emoji-tree per-source stats footer.

2 · The honest Google Trends constraint

You can't get "more focused" data out of Google Trends keylessly. The live feed trends.google.com/trending/rss?geo=XX is geo-only — I just probed it: &category= and &hours= are ignored, it always returns the same ~10 national breakout searches. The richer endpoints (interest-over-time, related queries, by-category) are the ones Google locked down; pytrends is archived/broken, and the only reliable granular path is paid (SerpApi / DataForSEO).

So "focused trend research" doesn't come from a better Trends API — it comes from treating the Trends board as one demand signal and corroborating the topic across other keyless sources, which is exactly what last30days does. That's the design's load-bearing idea.

3 · The recrafted skill — capability

Given a topic + an intent in the prompt, fuse the Google Trends demand signal (what people are searching) with a keyless multi-source pulse (what people are saying — Reddit, HN, YouTube, recent web), score & cluster it, and return a brief whose depth and format match the ask — from a 10-second board scan to a deep cited report to a Google Sheet on a timer.
1 · FetchGoogle Trends board (geo) + keyless fan-out scoped to the topic: Reddit RSS, HN Algolia, YouTube (yt-dlp), WebSearch recent news.
2 · Score + clusterDeterministic: recency + per-source engagement weighting → cluster corroborated items into trend "stories", tag single-source / thin-evidence honestly.
3 · PackageLLM writes the brief in the shape the intent selected; honest when a topic isn't on the board but is hot elsewhere.
