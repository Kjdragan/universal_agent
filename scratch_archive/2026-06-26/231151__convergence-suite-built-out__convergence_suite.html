How overlap is resolved — the downward-deferral chain

Each master's description defers to the leaner tool when it fits, so the router never double-claims a phrasing.

converge ──defers──▶ pass-fail-repair  you know it's a runnable check
           ──defers──▶ autoresearch-loop  you know it's a single number
           ──defers──▶ judge-panel  pure subjective quality
           ──defers──▶ optimize-then-verify  explicit "optimize-then-verify" structure

converge-flow ──defers──▶ converge  one artifact, not a sequence

How it's built

Engine-injection composition (there is no skill-to-skill API): a master imports the primitive engines and injects them. Every skill is built and verified in its own demo-* repo with a deterministic offline gate plus, where relevant, a real ZAI/GLM path — then promoted into the dragan plugin. All repos are born dark-factory (a self-contained auto-merge.yml validates and squash-merges every PR with no review).
