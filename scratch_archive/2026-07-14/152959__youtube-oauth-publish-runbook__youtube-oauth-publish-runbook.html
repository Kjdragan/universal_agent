Kill the weekly YouTube OAuth re-auth for good

Publish the Google OAuth app Testing → In production. One-time, ~5 clicks, only you can do it.  ·  2026-07-14

The honest status: We didn't actually publish the app — we built the workaround (the daily youtube_oauth_watchdog cron + the one-tap re-auth email you keep getting) and documented the permanent fix in two places, but never performed it. The emails still arriving is the proof the app is still in Testing mode. The real fix below takes about a minute and stops them permanently.

Why this happens

Google expires refresh tokens issued by an OAuth app whose publishing status is "Testing" after exactly 7 days — every time, no exceptions. Our token is minted, the digest runs for ~6 days, the watchdog emails you a re-auth button before day 7, you re-mint, repeat. The 7-day clock is tied only to the Testing publishing status, not to anything about our code. Moving the app to "In production" makes refresh tokens long-lived (valid until revoked, or 6 months unused).

The fix — publish to production

  1. Open Google Cloud Console and select the right project.
    The project that owns the YouTube OAuth client — its project number is the leading digits of YOUTUBE_OAUTH_CLIENT_ID (the part before the first -). Make sure that project is selected in the top-bar project picker.
  2. Go to the Auth Platform → Audience tab.
    APIs & Services → Google Auth Platform → Audience
    The old “OAuth consent screen” menu item now redirects here (Google reorganized it into Branding / Audience / Clients tabs).
  3. Click PUBLISH APP.
    Publishing status will read Testing. The button flips it to In production. Confirm the “push to production” dialog.
  4. Ignore the verification prompt.
    Google will invite you to submit for verification (because youtube is a restricted scope). You do not need to complete it. The 7-day expiry is tied to publishing status, not verification — it's already gone the moment status is "In production."
  5. Do one final re-auth to mint a fresh long-lived token.
    Tokens minted while in Testing may keep their 7-day fuse. Tap the button in the next watchdog email once (or run uv run python -m universal_agent.scripts.youtube_oauth2_setup). On the consent screen you'll see a one-time “Google hasn't verified this app” warning → Advanced → Go to Universal Agent (unsafe). Approve once. Done — that token is now long-lived and the emails stop.

Two birds: Publishing status is set per Google Cloud project (per consent screen), not per OAuth client. The Gmail/gws email auth has the exact same 7-day-Testing problem. If the YouTube client and the Gmail client live in the same project, this one publish also ends the weekly gws re-auth. If they're in different projects, repeat the steps for each. Check by comparing the leading project-number digits of the two client IDs — same number = same project.
