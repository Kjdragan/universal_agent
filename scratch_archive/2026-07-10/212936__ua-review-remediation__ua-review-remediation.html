Deliberately not changed

Fail-open ops auth — by design

_require_ops_auth intentionally fails open when no ops token is configured (tailnet-trust). The dangerous sudo systemctl activity surface is already fail-closed on the vps profile via _require_ops_auth_for_sudo_surface. Left as-is — not an oversight.

Least-privilege env for the spawned claude CLI — deferred (needs care)

The correct fix is a minimal env allow-list instead of full os.environ, but the CLI and its stdio MCP servers need many ${VAR}s resolved from .mcp.json; too narrow a list breaks Cody/VP execution. Recommended follow-up: enumerate the MCP-required vars from .mcp.json, build the allow-list, and test against a real Cody run before shipping. (The Link/Stripe bridge is a trusted deliberate exception.)
