What it built

A self-correcting loop. Instead of one-shot prompting, it drives a model in a loop: generate code → grade it by running a real pytest suite → feed the failures back → revise → repeat until the tests pass (or a round cap). The "loops, not prompts" idea — made trustworthy because the grader is real test execution, not the model judging itself.

Three tasks, each with a hidden test suite: nth_fibonacci to_iso_date bounded_stack. Real GLM calls (glm-5-turbo via ZAI), real test runs, no mocks. At least one task is nudged to miss on the first attempt so you can see the loop repair it.

How it got here — the autonomous pipeline

You type /demo <a link>. The factory (with Claude as the conductor) did all of this on its own:

  1. Classified the seed — a YouTube video on "Claude Code works better with loops."
  2. Checked the vault — "have we built this before?" No → build fresh.
  3. Researched the idea until it understood it well enough to write a spec.
  4. Picked the archetype — a framework/technique → a straight script.
  5. Scaffolded its own repo — ~/lrepos/demo-self-correcting-loop, wired to secrets, with the verifier gate installed.
  6. Built it autonomously — a headless /goal session wrote the code and re-ran the verifier until it passed.
  7. Landed it — wrote a manifest + a vault entry so the factory remembers it.

Run it yourself

It's a standalone repo on this machine. Two lines:

cd ~/lrepos/demo-self-correcting-loop
uv run python main.py

It bootstraps the factory's secrets itself (ZAI routing), runs the 3 tasks through the loop, prints the round-by-round trace, and exits 0 only if every task converges. To run the gate that judges it: uv run python scripts/verify_demo.py.

Note: it makes real GLM calls, so a full run takes a few minutes (model latency), and needs the desktop's Infisical creds (already present here).

Bonus — the run also closed a recursive-improvement loop

This first live build surfaced two real weak spots in the factory's heuristics, which were then fixed at the source using the captured build log as evidence:

WeaknessFix
Archetype guessed from the URL string — a YouTube URL has no "claude" in it, so it mis-defaulted.Classifier now takes researched context + reports a confidence; low-confidence URLs get decided from research, not the string.
False-positive "reuse" — over a tiny vault, the loop idea wrongly "matched" the unrelated deep-research demo on one weak shared word.Added a relevance gate (min shared meaningful tokens + coverage). Loop → build-new; unrelated → no match; real deep-research → still matches.

Both verified against the live vault; selftest 43/43 + regression 3/3 green.
