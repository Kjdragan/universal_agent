Model right-sizing of the demo_factory pipeline

Goal: use the cheapest model that holds demo quality at each pipeline stage — trim Opus (which bills the Claude-Code / Anthropic-Max plan) to Sonnet/Haiku where they pass the gates — chosen per demo class. Config + analysis, not a rebuild.

The seams (where models are chosen)

Stage Knob Default
Build (/goal agent, real Anthropic Max) goal_build.py --model / build_demo.py --build-model ← _launch.goal_argv(model=) claude-opus-4-8
Fidelity judge (Gemini) evaluate_artifact.py --model / land_demo.py --eval-model / build_demo.py --eval-model gemini-3-flash-preview

The deterministic verifier (verify_demo.py) is model-free — left untouched. The /goal build must stay on real Anthropic Max (its internal evaluator is real Haiku); it is never routed to ZAI (_launch.py scrubs ANTHROPIC_* and forwards the OAuth token).

Design

  • scripts/model_tiers.py — maps a demo class → {build, eval, eval_samples}.
  • Build models are Claude aliases (opus/sonnet/haiku), not pinned ids. Per code.claude.com/docs/model-config the --model flag accepts these aliases and resolves them to the current model on the Anthropic-Max path (opus→Opus 4.8, sonnet→Sonnet 4.6, haiku→current Haiku). No stale version ids to rot, and "the haiku tier" is exactly the right-sizing intent.
  • Class is derived from source_detect.py (archetype_hint + type): agent_sdk (Claude ecosystem), visual (image/youtube), else generic.
  • build_demo.py --model-tier auto — opt-in; resolves build+eval per class. Default is fixed (uses --build-model/--eval-model as given), so nothing changes until you opt in.
  • One-flag fallback: DEMO_FACTORY_ALL_OPUS=1 forces every build back to opus. Per-class / global overrides: DEMO_FACTORY_BUILD_MODEL_<CLASS>, DEMO_FACTORY_EVAL_MODEL.
  • scripts/model_rightsize_eval.py — the measurement harness (reproduces the evidence).

Evidence — generic class (Mandelbrot study)

Seed: "render a Mandelbrot set PNG with numpy and pillow", live --build-only builds on the VPS (real Anthropic Max), scored by the deterministic verify gate (BUILD_DEMO: PASS == verify ran=1 exit=0). 2026-06-30.

build tier verify gate wall time
opus PASS 179 s
sonnet PASS 182 s
haiku PASS 98 s

All three tiers produced a working, verified demo. Haiku was ~45% faster and is the cheapest viable tier for this simple seed.

Decision (current config)

class build rationale
generic sonnet Right-sized down from Opus. Sonnet passes the gate at ~Opus speed; safe across the class's complexity range on current (1-seed) evidence. Opus→Sonnet is the cost win for the common proactive class.
agent_sdk opus Conservative — Claude-ecosystem demos are harder; not yet measured.
visual opus (+5 fidelity votes) Conservative — multimodal fidelity is harder to judge; not yet measured.

Next step (more savings): Haiku cleanly built + verified the simple Mandelbrot demo and was fastest. With 2–3 more generic seeds confirming, split a simple/local-compute sub-class → haiku (or just set DEMO_FACTORY_BUILD_MODEL_GENERIC=haiku). One seed isn't enough to send the whole generic class to Haiku — a complex generic API demo may need more.

Note: this study scored the verify gate (it ran + produced an artifact). The harness also supports the Gemini fidelity judge (--eval) for a fuller quality signal; re-run with --eval on richer seeds when right-sizing the harder classes.

How to extend

python3 scripts/model_rightsize_eval.py --seed "<seed>" --demo-id <id> \
    --tiers opus,sonnet,haiku --workspace-root /tmp/rs [--eval] --out rs.json

Then update scripts/model_tiers.py::_TIERS for the class from the recommendation.

Not done here (gated)

Flipping the proactive default to --model-tier auto (so autonomous demos right-size by class) is a deliberate behavior change — left for operator approval. It's a one-line edit to the proactive objective in universal_agent/services/cody_dispatch.py (add --model-tier auto to the build_demo.py invocation). Until then, /demo and proactive both stay on the current default (Opus); the right-sizing is available via --model-tier auto on demand.
