Does our loop system beat bare /loop?

A controlled A/B on glm-5.2, graded by an objective arbiter (regex/length + a blind independent grader), never the loop's own judgment. The investigation resolved to a single mechanism. 2026-06-22.

Answer: yes — conditionally, and not for the reason we guessed. On a vague task (no operator-supplied criteria), turning the goal into an explicit, externalized checklist the artifact is answered against more than doubled the objective score (0.31 → 0.75) for ~1 extra model call. The independent judge added only a small top-up (→ 0.875) at 3× the cost. On tasks where you already wrote a sharp check, bare /loop ties everything.

The mechanism (the one thing that does the work)

Externalize the check. Turn the goal into an explicit, separate checklist and answer against it. A check you write down binds the model; a check it only recites to itself (derive-inline) does not — that's the entire difference between 0.31 and 0.75. It works whether you write the checklist or the model derives it as a separate step.
  • Deriving good criteria was never the problem. Told to, glm-5.2 derived essentially the grading rubric (avoid these buzzwords, be concrete, name an audience, be concise). It just ignored its own criteria when they lived inline — in one trial it listed "remove these buzzwords" and then emitted the original buzzword-laden sentence verbatim and declared DONE.
  • The independent judge is a modest top-up, not the lever. It added 0.75 → 0.875 at 3× the calls, and in these runs it never even fired a correction (the externalized checklist already got the author there).

How we got here (and two confounds we caught)

stepfinding
Pass 1 (judge)improved tied bare everywhere — judge looked useless.
Confound #1The one apparent win (N2 0.25→1.00) was information: bare wasn't told the criteria. Told them, bare hit 1.00 too.
Discipline-aloneOn a vague task, deriving criteria inline + self-verifying did nothing (0.31 = bare). So "the discipline helps" was also unproven.
derive + judgeDeriving criteria separately + enforcing → 0.875. The system flipped to a clear win on vague tasks.
Confound #2 / isolationThe judge never corrected anything. Isolating it (derive_explicit, no judge) → 0.75. So the lever is externalizing the check, not the judge.

What we changed, and honest limits

The /loop discipline was corrected to "derive the check as a separate explicit checklist, then answer against it" — the 0.75 version — instead of the original "derive + self-verify" (the 0.31 version that didn't work). The independent judge is demoted to an optional top-up.

Limits: one model (glm-5.2); the resolving ladder is one task family (a vague pitch), 4 trials/cell; the "externalize the check" effect is intuitive and likely general but is so far shown on this task. The judge's value may be larger on harder tasks where the author can't get it right first try (untested), or with a different-model judge (we had no real Anthropic key, so cross-model was out of reach).
