4. The cumulative arc — six threads ↑

Read release-by-release it looks like noise. Read as threads, the whole 1.6.7 → 1.7.6 window has a clear direction: move the work off the cloud, stop losing the user's words, and make Linux a real platform rather than a port.

① Local & private inference went from possible to default

The single strongest thread. It starts with self-hosted endpoints being allowed and ends with GPU-accelerated streaming decode running on-device by default.

1.6.8 self-hosted + local live preview → 1.7.0 per-scope LLM config + Parakeet → 1.7.4 Tinfoil enclaves + Azure → 1.7.5 Corti reasoning, OpenRouter → 1.7.6 Vulkan GPU, Nemotron streaming, LFM2 local models

② "Never lose the user's words" became an explicit invariant

Early releases fixed individual data-loss bugs. Later ones added structural fallbacks: every failure path now degrades to pasting something rather than nothing.

1.6.9 retry button on failures → 1.7.3 discarded dictations saved → 1.7.4 empty recordings stop crashing → 1.7.6 failed cleanup pastes raw + toast; failed translation pastes untranslated; GPU crash restarts on CPU and retries the same request

③ Credentials and auth got hardened, then audited

A one-release rewrite followed by sustained tightening — the language of the later notes (SOC 2, lockfile-integrity CI guards, attested enclaves) says they're selling into enterprises now.

1.7.0 keys + sessions to OS keychain, Better Auth → 1.7.2 zero spurious Keychain prompts → 1.7.4 SOC 2 dependency hardening + Enterprise SSO → 1.7.5 Bedrock/Corti compliance scopes

④ Linux stopped being a second-class target

Nine of the ten releases carry Linux-specific fixes. The arc moves from compositor coverage (making hotkeys work at all across desktop environments) to distribution correctness (packaging that doesn't destroy your data).

1.6.6 KDE Wayland → 1.6.8 Hyprland + GNOME conflicts → 1.6.9 evdev push-to-talk → 1.7.1 COSMIC, Sway/i3, terminal paste → 1.7.2 Wave Terminal allowlist → 1.7.4 openSUSE + hardened kernels → 1.7.5 Wayland/Node 24 crash → 1.7.6 .deb maintainer scripts

⑤ Latency and CPU cost fell, structurally

Not tuning — architecture. The chunked record-then-transcribe model was replaced by a persistent stream that commits on stop.

1.6.8 live preview (still chunked) → 1.7.1 silence/breathing gating → 1.7.6 persistent stream, single-pass commit, ~half the CPU per dictation, bounded llama context

⑥ The product grew from dictation tool into a notes/meetings workspace

Worth naming because it's most of the changelog volume and least relevant to you — if you only dictate into an editor, roughly half of every release since 1.6.6 has been features you never touch.

1.6.6 rich text → 1.6.7 chat + semantic search → 1.6.9 sync + export → 1.6.10 integrations hub → 1.7.3 snippets + dictionary → 1.7.6 collapsible sidebar, URL/batch import

6. What this means for dictating code ↑

Filtering all ten releases down to the workflow you actually use — speech into a terminal and editor on Linux — here's what changed and what's now available that you may not have turned on.

Already working better than when you started

  • Cleanup stopped answering you. 1.7.4 fixed the failure mode where dictated text that sounded like a question got answered by the cleanup model instead of tidied. Directly relevant when you dictate prompts.
  • Voice agent only fires when addressed. Same release — dictations only route to the agent when you actually say its name.
  • Failures paste something. As of 1.7.6, a failed cleanup pastes your raw transcript with a toast. You no longer lose a long dictation to a provider hiccup.
  • Mic fallback. A muted or vanished device falls back to the default instead of silently recording nothing (1.7.4, hardened in 1.7.6) — and your selection now survives Chromium's device-ID rotation.
  • Half the CPU per dictation, if you move to a streaming model (1.7.6).

Available and worth a look

  • Snippets (1.7.3). Spoken shortcut → pasted boilerplate. The obvious win for repeated code/prompt fragments.
  • Multiple hotkeys per action (1.7.5). Bind dictation to more than one key.
  • Nemotron streaming EN 0.6B (1.7.6, 632 MB). Live text as you speak plus single-pass commit. Your 1660 Ti has 6 GB — this is small and fits comfortably alongside your Qwen3.5-4B cleanup model.
  • LFM2.5 230M / 1.2B (1.7.6, 0.25–0.73 GB). If your 3 GB Qwen3.5-4B cleanup model feels heavy on a 6 GB card, these are built for exactly that constraint.
  • The custom dictionary (redesigned 1.7.3) is where you teach it your library and identifier names — the highest-leverage setting for dictating code, and it syncs across devices now.

One caveat on the GPU items: the local-LLM freeze bug fixed in 1.7.6 (unbounded llama-server context) was reported hardest on unified-memory Macs, but the same unbounded allocation applied everywhere. If local cleanup with your Qwen3.5-4B ever felt like it hung the machine before today, that's the explanation — and 1.7.6 is the fix.
