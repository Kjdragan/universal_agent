Scratchpad markup, upgraded try it below

This is a hand-authored HTML exhibit — exactly the kind of page that used to have no review toolbar. Now it does. Tap the 💬 Review button at the bottom-right and try the three new things below, then Submit to copy a [scratch-review] prompt back to me.

1 · Precise text anchoring

Pick a comment lands exactly where you meant it. Pick a comment used to land on the first match of identical text; now each comment records a stable element selector plus an occurrence index. The phrase pick a comment appears several times in this paragraph — highlight the last one and hit + Comment; the prompt I receive will say [occ 3], not occurrence 1.

Try: highlight any sentence above → a 💬 Comment chip appears → click it → type a note. The highlighted text stays marked on the page.

2 · Annotate non-text elements

Diagrams, images, tables and charts have no text to select — so there's a new + Element mode. Tap it, then tap the element you mean.

ActionWhat it anchors to
+ Commentselected text + selector + occurrence index
+ Elementthe tapped element's selector (e.g. #hero)
+ Notenothing — a general remark
Try: + Element → tap the diagram or a table cell. The element gets a dashed outline and the comment records its selector.

3 · Open-time layout audit

When a page opens, the toolbar quietly checks for real render defects — the page overflowing the screen horizontally, an element pushed past the viewport, or a Mermaid diagram that failed to render. If it finds any, a banner offers to add them to your review round, and they're written into the downloaded JSON either way. (This page is clean, so you shouldn't see the banner.)

How the round comes back to me

Submit downloads scratch-comments-<slug>.json and copies a prompt to your clipboard. Paste it into our chat — I read each comment with its location and anchor, re-read this exhibit, apply the changes, and re-publish in place. Same loop as before, just sharper.
