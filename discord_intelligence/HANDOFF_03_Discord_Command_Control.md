## Purpose

Transform the owner's existing (currently empty) Discord server into a structured operational interface for the Universal Agent. This provides a richer command & control experience than Telegram or email, with persistent organized channels, real-time slash commands, rich embed responses, and cross-integration with the Intelligence Daemon's research feed.

## Slash Commands

All commands are registered as Discord application commands (slash commands) with autocomplete and parameter validation.

### Operations Commands

| Command | Description | Maps To |
|---------|------------|---------|
| `/status` | Show UA system overview — running missions, heartbeat health, queue depth | Gateway Ops API |
| `/briefing [now\|morning\|weekly]` | Trigger or retrieve briefing | Proactive Pipeline |
| `/task add <description> [priority]` | Create Task Hub item | Task Hub API |
| `/task list [status] [limit]` | Show task queue | Task Hub API |
| `/mission status [id]` | Show active mission details | VP Delegation |
| `/mission list` | List all missions with status | VP Delegation |

### Intelligence Commands

| Command | Description | Maps To |
|---------|------------|---------|
| `/research <topic>` | Commission a research mission on a topic | ATLAS VP delegation |
| `/discord search <query> [server] [channel]` | Search ingested Discord messages | discord_intelligence DB |
| `/discord signals [type] [since]` | Show recent detected signals | signals table |
| `/discord insights [limit]` | Show top unread insights | insights table |
| `/wiki query <question>` | Query the LLM Wiki | LLM Wiki System |
| `/wiki add <title> <content>` | Add entry to wiki | LLM Wiki System |

### System Commands

| Command | Description | Maps To |
|---------|------------|---------|
| `/monitor add <channel_id> <tier>` | Add a channel to monitoring | daemon config |
| `/monitor remove <channel_id>` | Remove a channel from monitoring | daemon config |
| `/monitor list` | Show all monitored channels | daemon config |
| `/config triage-frequency <hours>` | Adjust Layer 3 processing frequency | triage scheduler |

## Integration with Intelligence Daemon (Tier 1 × Tier 2)

The Command & Control server's INTELLIGENCE channels are populated by the Intelligence Daemon:

### #announcements-feed
Every Layer 2 `announcement` and `release` signal gets auto-posted here as a rich embed:

```python
async def post_signal_to_cc_server(signal):
    """Post a Layer 2 signal to the Command & Control server's announcements feed."""
    channel = client.get_channel(ANNOUNCEMENTS_FEED_CHANNEL_ID)
    
    embed = discord.Embed(
        title=f"📢 {signal['type'].title()}: {signal['data']['server']}",
        description=signal['data']['preview'][:2000],
        color=discord.Color.blue() if signal['severity'] == 'info' else discord.Color.orange(),
        timestamp=signal['created_at']
    )
    embed.add_field(name="Source", value=f"#{signal['data']['channel']}", inline=True)
    embed.add_field(name="Server", value=signal['data']['server'], inline=True)
    
    await channel.send(embed=embed)
```

### #research-feed

## Simone Chat Integration

The `#simone-chat` channel deserves special attention. This is where the owner can have a direct, real-time conversation with Simone — similar to email/AgentMail but immediate.

### How It Works

Messages in `#simone-chat` from the owner get routed to Simone's processing pipeline. Simone's responses get posted back to the channel. This creates a persistent, searchable conversation history organized by channel rather than email threads.

```python
@client.event
async def on_message(message):
    if message.channel.id == SIMONE_CHAT_CHANNEL_ID:
        if message.author.id == OWNER_DISCORD_ID:
            # Route to Simone
            response = await route_to_simone(message.content)
            
            # Format and send response
            embed = discord.Embed(
                description=response['text'],
                color=discord.Color.teal(),
                timestamp=datetime.utcnow()
            )
            embed.set_author(name="Simone", icon_url=SIMONE_AVATAR_URL)
            
            if response.get('artifacts'):
                for artifact in response['artifacts']:
                    embed.add_field(
                        name=f"📎 {artifact['name']}",
                        value=artifact['url'],
                        inline=False
                    )
            
            await message.channel.send(embed=embed)
```

### Thread-Based Mission Tracking

When a mission is created (either by command or by intelligence trigger), a thread is auto-created in `#mission-status`:

```python
async def create_mission_thread(mission):
    channel = client.get_channel(MISSION_STATUS_CHANNEL_ID)
    
    # Create the initial message
    embed = discord.Embed(
        title=f"🚀 Mission: {mission['title']}",
        description=mission['description'],
        color=discord.Color.blue()
    )
    embed.add_field(name="Status", value="📋 Created", inline=True)
    embed.add_field(name="Assigned To", value=mission.get('delegated_to', 'Unassigned'), inline=True)
    embed.add_field(name="Priority", value=mission.get('priority', 'medium'), inline=True)
    
    msg = await channel.send(embed=embed)
    
    # Create a thread on the message
    thread = await msg.create_thread(
        name=f"Mission: {mission['title'][:90]}",
        auto_archive_duration=10080  # 7 days
    )
    
    # Post updates to the thread as mission progresses
    return thread.id
```

## Implementation Notes for Agent (Claude Code / Other)

1. **The Command & Control bot can be the SAME bot process as the Intelligence Daemon.** Don't create a second bot. Extend the daemon from HANDOFF_02 with the slash commands and CC server functionality.

2. **Slash commands need to be synced**: After adding commands, call `bot.tree.sync()` to register them with Discord. For development, sync to a specific guild (instant) rather than globally (takes up to an hour).

3. **Check the Telegram integration pattern**: The existing Telegram bot in the UA codebase (`docs/03_Operations/91_Telegram_Architecture_And_Operations_Source_Of_Truth_2026-03-06.md`) shows how the UA routes messages from chat interfaces to agent processing. Follow the same pattern for Discord.

4. **Owner's Discord server has no external users**: The bot has full control. No need to worry about permissions for other users, moderation, etc.

5. **The owner is already using email/AgentMail with Simone**: The Discord `#simone-chat` should feel like an alternative channel to the same Simone, not a different agent. Same personality, same capabilities, just a different interface.

6. **Survival of the fittest**: The owner wants overlapping channels (email, Telegram, Discord) and will let natural preference determine which survives. Don't deprecate anything — add Discord as an option.
