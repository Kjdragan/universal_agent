# Phase 4 Consolidation Notes — Salvage Inventory

**Purpose.** Before the legacy `docs/` corpus (214 files) is archived, this note inventories
still-valuable, still-true knowledge that is **not** already captured by a canonical doc's
`legacy_refs`. Each row states the fact, cites the legacy source, names the canonical doc that
should absorb it (by manifest `id`), and marks `absorb` vs `archive-only`.

**Method.** Skimmed the 48-doc manifest, then scanned the soon-to-be-archived strata the
canonical set does *not* already point at: `docs/03_Operations/` dated audit/handoff reports,
`docs/operations/` dated files, `docs/proactive_signals/` dated files, `docs/03_Run_Reviews/`,
`docs/reports/`, `docs/05_Archive/`. Selection bar: an operational fact, hard-won lesson, design
rationale, or recurring-incident pattern that (a) is still true against current code/infra and
(b) is genuinely additive to the canonical doc that owns the area. Pure point-in-time status
(soak readiness, "Phase N shipped", run-review session logs) is marked `archive-only`.

Most rows below carry **rationale or environmental/operational knowledge** — the kind the
governance rules (`02_GOTCHA_INVENTORY.md`) explicitly want preserved — not code-behavior claims
the source code already arbitrates.
