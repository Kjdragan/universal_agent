# Hermes Ship 5 — Task Hub Observability Protocol allowlist.
#
# Pre-existing subprocess-spawning files in src/universal_agent/ that
# predate the protocol shipped in Ship 4. Each entry MUST justify
# why the file is on the list. Two acceptable rationales:
#
#   (a) "Not a task unit" — the spawn is infrastructure (CLI launcher,
#       Playwright runtime, PDF extractor, OAuth bridge, GWS calendar
#       fetch, dependency sweep). The work it performs is not a
#       Task Hub entity; there is no task_hub_items row to wire to.
#       The protocol doesn't apply.
#
#   (b) "Legacy task unit, follow-up wiring planned" — the spawn IS
#       part of a task lifecycle and should eventually be wired. The
#       entry stays here until a follow-up PR migrates it.
#
# Adding a NEW entry to this allowlist requires explicit operator
# approval. The protocol is the long-term standard; allowlist exists
# only to make adoption incremental, not to permit drift.
#
# Removing an entry happens automatically when the file becomes
# compliant (imports a protocol helper). The lint guard warns on
# stale entries but doesn't fail — operator cleans them up in a
# follow-up PR.
#
# See docs/03_Operations/129_Task_Hub_Observability_Protocol.md
# for the rules this allowlist gates.
